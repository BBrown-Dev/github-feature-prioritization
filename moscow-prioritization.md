# MoSCoW Prioritization Model for GitHub Enhancements

## MUST-HAVE
- Improved GitHub Actions debugging and clearer error logs
- Better performance for large repositories and monorepos
- Reliable sync between web, desktop, and CLI
- Fix automation reliability issues (Actions & Dependabot)
- Enhanced search and filtering for large Java codebases

## SHOULD-HAVE
- Issue and PR dependency tracking (blockers & prerequisites)
- Basic built-in reporting (commit frequency & PR cycle time)
- Improved integration stability (Slack, Jira, & cloud providers)
- More granular permissions for GitHub Actions

## COULD-HAVE
- Visual pipeline editor for GitHub Actions
- Offline mode improvements
- Customizable UI themes
- AI-assisted PR review summaries

## WON'T-HAVE (for now)
- Full project portfolio management
- Built-in time tracking
- Native Kubernetes deployment dashboard
- Full IDE replacement
